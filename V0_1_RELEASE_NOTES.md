# V0.1 Release Notes

## 发布概览
V0.1 聚焦“主流程可联调 + 异常可回退 + 验收可执行”。
本版本在代码层新增了章节流程模拟器，用于从新建战役到第一章结算的端到端验证，并补齐异常演练、验收清单、演示脚本与演示存档。

## 本版本完成内容

### 1. 端到端联调（新建战役 -> 第一章结算）
- 新增 `CampaignSimulator`，覆盖开局建档、章节推进、结算落档完整链路。
- 产出统一结果对象：事件序列、结构化日志、性能指标、checkpoint 信息。

### 2. 异常场景演练与回退
- 模型超时：自动使用文本模板回退。
- 图片失败：自动使用占位图回退。
- 存档中断：自动回滚到最近 checkpoint，并补偿完成章节结算。

### 3. 验收体系
- 功能验收：流程走通与结算状态。
- 日志验收：关键步骤日志 + 异常原因可追踪。
- 性能验收：首屏时间与单步响应时间已纳入输出口径。

### 4. 演示资产
- 已固化 10-15 分钟演示脚本。
- 已固化“演示战役存档”：`data/demo_campaign_save_v0_1.json`。

## 验证结果（本地）
- 新增测试覆盖：
  - E2E 主流程成功。
  - 三类异常联动回退成功。
  - 验收指标字段存在。
  - 演示存档可复用且处于结算完成态。

## 已知限制
- 当前为流程模拟器形态，未接入真实 LLM / 图像模型网络调用。
- 未集成桌面端容器（Tauri/Electron）与 UI 自动化。
- 性能指标为逻辑链路基线，不代表真实生产负载。

## 下一阶段风险清单（V0.2 关注）
1. **真实模型延迟抖动风险（高）**
   - 风险：在线推理时延显著高于模拟环境，回退触发频率可能上升。
   - 缓解：接入超时分层、指数退避与 provider 级熔断；建立 p95/p99 监控。
2. **存档一致性风险（高）**
   - 风险：真实落盘与事件写入可能出现部分成功，导致读档状态不一致。
   - 缓解：引入“状态+事件”事务写入与恢复日志（WAL/补偿事务）。
3. **异常恢复可观测性不足（中）**
   - 风险：回退发生后若缺少链路追踪，不利于问题复盘。
   - 缓解：增加 trace_id/span_id，将回退前后状态 hash 入日志。
4. **演示存档与真实环境漂移风险（中）**
   - 风险：演示用例稳定但无法代表真实运行复杂度。
   - 缓解：每迭代追加 1 个“真实模型驱动”金丝雀战役样本。
5. **性能预算失控风险（中）**
   - 风险：接入真实服务后首屏和单步响应可能超预算。
   - 缓解：定义预算阈值（首屏/单步/p95），在 CI 加入性能门禁。
